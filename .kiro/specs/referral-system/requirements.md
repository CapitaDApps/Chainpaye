# Requirements Document: Referral System

## Introduction

This document specifies the requirements for a referral system integrated into a WhatsApp-based crypto payment application. The system enables users to earn rewards by referring new users, with earnings based on transaction fees generated by their referrals during a 30-day period.

## Glossary

- **Referral_System**: The complete referral tracking and rewards system
- **User**: A registered application user who has completed KYC
- **Referrer**: A user who has shared their referral code with others
- **Referred_User**: A user who registered using another user's referral code
- **Referral_Code**: A unique identifier assigned to each user for sharing
- **Point**: A unit of value where 1 point equals 1 USD
- **Offramp_Transaction**: A transaction converting cryptocurrency to fiat currency
- **KYC**: Know Your Customer verification process
- **Referral_Relationship**: The immutable connection between a referrer and referred user
- **Referral_Period**: The 30-day window during which referral earnings are active
- **Withdrawal_Request**: A user's request to transfer points to their bank account

## Requirements

### Requirement 1: Referral Code Generation

**User Story:** As a verified user, I want to receive a unique referral code automatically, so that I can share it with others and earn rewards.

#### Acceptance Criteria

1. WHEN a user completes KYC successfully, THE Referral_System SHALL generate a unique referral code for that user
2. THE Referral_System SHALL ensure all generated referral codes are unique across all users
3. THE Referral_System SHALL create referral codes that are alphanumeric and between 6-12 characters in length
4. WHEN a referral code is generated, THE Referral_System SHALL persist it to the user's record immediately

### Requirement 2: Referral Code Capture and Validation

**User Story:** As a new user, I want to enter a referral code before signup, so that I can be connected to the person who referred me and see a personalized invitation.

#### Acceptance Criteria

1. WHEN a new user types "start [referral_code]", THE Referral_System SHALL validate that the referral code exists in the system
2. WHEN a valid referral code is provided, THE Referral_System SHALL store the referral code in Redis with a 24-hour expiration linked to the user's phone number
3. WHEN a valid referral code is provided, THE Referral_System SHALL retrieve the referrer's name and display a personalized message: "You have been invited to join ChainPaye by {referrer_name}"
4. WHEN an invalid referral code is provided, THE Referral_System SHALL reject the code and notify the user with an appropriate error message
5. AFTER displaying the invitation message, THE Referral_System SHALL proceed with the normal signup call-to-action flow

### Requirement 2.1: Referral Code Integration in Signup Flow

**User Story:** As a new user who entered a referral code, I want the code to be automatically filled during signup but still be editable, so that I have control over my referral connection.

#### Acceptance Criteria

1. WHEN a user begins the signup process, THE Referral_System SHALL check Redis for any stored referral code associated with the user's phone number
2. IF a referral code exists in Redis, THE Referral_System SHALL pre-populate the referral code field in the signup form
3. THE Referral_System SHALL make the referral code field optional and editable during signup
4. WHEN the user completes signup with a referral code, THE Referral_System SHALL validate the code again before creating the relationship
5. WHEN a valid referral code is confirmed during signup, THE Referral_System SHALL create an immutable referral relationship between the referrer and referred user
6. WHERE a user has already been referred, THE Referral_System SHALL prevent any modification to their existing referral relationship
7. WHEN a user attempts to use their own referral code, THE Referral_System SHALL reject the self-referral

### Requirement 3: Referral Earnings Calculation

**User Story:** As a referrer, I want to earn 25% of transaction fees from my referred users, so that I am rewarded for bringing new users to the platform.

#### Acceptance Criteria

1. WHEN a referred user completes an offramp transaction, THE Referral_System SHALL calculate the transaction fee as 1.5% of the transaction amount
2. WHEN a transaction fee is calculated, THE Referral_System SHALL compute referrer earnings as 25% of that fee
3. WHILE a referred user is within their 30-day referral period, THE Referral_System SHALL credit earnings to the referrer's point balance
4. WHEN a referred user exceeds their 30-day referral period, THE Referral_System SHALL stop crediting earnings to the referrer
5. THE Referral_System SHALL maintain the referral relationship permanently even after the 30-day earning period expires

### Requirement 4: Point Balance Management

**User Story:** As a user, I want my referral earnings tracked as points, so that I can accumulate rewards and request withdrawals.

#### Acceptance Criteria

1. THE Referral_System SHALL track points where 1 point equals 1 USD
2. WHEN referral earnings are generated, THE Referral_System SHALL add the points to the referrer's current balance
3. WHEN a withdrawal is processed, THE Referral_System SHALL deduct the withdrawal amount from the user's point balance
4. THE Referral_System SHALL maintain separate tracking of total earned points and current available balance
5. THE Referral_System SHALL ensure point balances never become negative

### Requirement 5: Withdrawal Request Processing

**User Story:** As a referrer with accumulated points, I want to withdraw my earnings to my bank account, so that I can access my rewards.

#### Acceptance Criteria

1. WHEN a user requests a withdrawal, THE Referral_System SHALL verify the user has at least 100 USD in their point balance
2. WHEN a user requests a withdrawal, THE Referral_System SHALL verify the user has not made a withdrawal request in the past 7 days
3. WHEN a valid withdrawal request is made, THE Referral_System SHALL create a pending withdrawal record
4. WHEN a withdrawal request is created, THE Referral_System SHALL initiate a 24-hour approval process
5. WHEN a withdrawal is approved, THE Referral_System SHALL deduct the amount from the user's point balance and initiate bank transfer
6. IF a withdrawal request fails validation, THEN THE Referral_System SHALL return an error message explaining the reason

### Requirement 6: Referral Dashboard Display

**User Story:** As a user, I want to view my referral statistics and earnings, so that I can track my referral performance.

#### Acceptance Criteria

1. WHEN a user types "referral", THE Referral_System SHALL display the user's unique referral code
2. WHEN displaying the dashboard, THE Referral_System SHALL show the user's referral link
3. WHEN displaying the dashboard, THE Referral_System SHALL show the total count of users referred by this user
4. WHEN displaying the dashboard, THE Referral_System SHALL show the current available point balance
5. WHEN displaying the dashboard, THE Referral_System SHALL show the lifetime total points earned
6. WHEN displaying the dashboard, THE Referral_System SHALL show the total offramp volume from all referred users
7. WHEN displaying the dashboard, THE Referral_System SHALL show the total fees generated from referred users' transactions
8. WHEN displaying the dashboard, THE Referral_System SHALL show the total earnings from referrals

### Requirement 7: Leaderboard Display

**User Story:** As a user, I want to see a leaderboard of top referrers, so that I can see how my performance compares to others.

#### Acceptance Criteria

1. WHEN a user accesses the leaderboard, THE Referral_System SHALL display users ranked by total points earned in descending order
2. WHEN displaying leaderboard entries, THE Referral_System SHALL show each user's total points earned
3. WHEN displaying leaderboard entries, THE Referral_System SHALL show each user's total count of referred users
4. THE Referral_System SHALL make the leaderboard accessible from the referral dashboard

### Requirement 8: Referral Period Tracking

**User Story:** As a system administrator, I want referral earnings to automatically stop after 30 days, so that the referral incentive structure is maintained correctly.

#### Acceptance Criteria

1. WHEN a referral relationship is created, THE Referral_System SHALL record the timestamp of relationship creation
2. WHEN processing an offramp transaction, THE Referral_System SHALL calculate the time elapsed since the referred user joined
3. WHEN the elapsed time is less than or equal to 30 days, THE Referral_System SHALL process referral earnings
4. WHEN the elapsed time exceeds 30 days, THE Referral_System SHALL skip referral earnings processing
5. THE Referral_System SHALL preserve the referral relationship data indefinitely regardless of the 30-day period

### Requirement 10: Temporary Referral Code Storage

**User Story:** As a system administrator, I want referral codes to be temporarily stored when users enter them via the start command, so that they can be retrieved during the signup process.

#### Acceptance Criteria

1. WHEN a user types "start [referral_code]", THE Referral_System SHALL store the referral code in Redis using the user's phone number as the key
2. THE Referral_System SHALL set a 24-hour expiration time for stored referral codes in Redis
3. WHEN retrieving a referral code from Redis during signup, THE Referral_System SHALL handle cases where the code has expired gracefully
4. WHEN a referral relationship is successfully created, THE Referral_System SHALL remove the temporary referral code from Redis
5. THE Referral_System SHALL ensure Redis storage operations do not block the user experience

### Requirement 9: Data Integrity and Validation

**User Story:** As a system administrator, I want the referral system to maintain data integrity, so that earnings and relationships are accurate and trustworthy.

#### Acceptance Criteria

1. THE Referral_System SHALL validate all referral codes before creating relationships
2. THE Referral_System SHALL prevent duplicate referral relationships for any user
3. THE Referral_System SHALL ensure all point calculations are performed with decimal precision
4. THE Referral_System SHALL persist all referral earnings transactions with timestamps and transaction references
5. THE Referral_System SHALL maintain an audit trail of all withdrawal requests and their statuses
